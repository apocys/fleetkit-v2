SpawnKit ‚Äî Platform Overview & E2E Test Plan

What is SpawnKit?

SpawnKit is the friendliest way to use AI agents. No terminal. No config files. No technical knowledge needed.

You get a pre-built AI team that works for you out of the box. A CEO agent that knows how to delegate, specialists that handle the work, and a visual dashboard where you can see everything happening ‚Äî like watching a little company run itself.

Think of it as: "Your personal AI company, ready in 60 seconds."

Who is it for?

  • Newcomers to AI ‚Äî people who want AI to help them but don't know where to start
  • Kids & teenagers ‚Äî gamified themes turn AI work into something fun (your CEO lives in a castle, your engineers work in a forge)
  • Non-technical users ‚Äî grandma can set it up. If you can use a phone, you can use SpawnKit
  • Power users too ‚Äî underneath it's real OpenClaw, so it scales to serious workloads

The core idea

OpenClaw is powerful but technical. SpawnKit wraps it in a visual, approachable layer:

  • Pre-built agent team ‚Äî no need to configure agents yourself. You get a CEO, CTO, COO, and more, ready to go
  • CEO is the brain ‚Äî you talk to the CEO, who orchestrates the whole team. The CEO is excellent at breaking down your request into tasks and delegating to the right specialist
  • Agents can spawn helpers ‚Äî when a task is complex, main agents spawn sub-agents to help. You see them appear in real-time
  • Missions ‚Äî your agents have a task board (like a to-do list, but collaborative). Multiple agents can work on the same mission together
  • Two built-in superpowers ‚Äî every team ships with two powerful skills: one for creative brainstorming/planning, and one for iterative refinement loops. These let your team tackle any problem from day one
  • Add new skills easily ‚Äî want your team to learn something new? Adding MCP connections or skills should feel as easy as installing an app
  • Connected fleets ‚Äî your SpawnKit instance (your "fleet") can connect to other fleets, like roads between cities. Some fleets might specialize in coding, others in research, others in creative work. They can help each other, share compute, or serve as backup

Why connected fleets matter

Imagine your fleet is great at writing but slow at coding. Your neighbor's fleet has powerful compute and a code-specialized team. Instead of doing everything yourself, your CEO sends the coding task to the other fleet ‚Äî like outsourcing to a specialist. Fleets can:

  • Specialize ‚Äî each fleet focuses on what it's best at
  • Share compute ‚Äî a fleet with more resources can help a smaller one
  • Backup ‚Äî if your fleet goes down, another can pick up the work
  • Collaborate ‚Äî two fleets working together on a big project

All Pages & What They Do

1. spawnkit.ai ‚Äî Landing Page

Purpose: Explain what SpawnKit is. Get people to try it.

  • Hero: "Your AI team, visualized" ‚Äî clear, non-technical pitch
  • Live preview: toggle between visual themes to see what the dashboard looks like
  • 24 use cases in plain language (email, research, content, coding, operations, personal tasks)
  • "How it works" ‚Äî 3 simple steps (Pick your style ‚Üí Meet your team ‚Üí Watch the magic)
  • Pricing: Free to start, Pro when ready
  • Testimonials from beta testers
  • Footer: GitHub, Discord, Docs

2. app.spawnkit.ai ‚Äî Theme Hub (entry point)

Purpose: Choose your dashboard style. This is the front door of the app.

  • 5 visual themes as cards: Executive (clean/professional), Green Iso (eco/nature), GameBoy (retro), Sims (playful), SimCity (city builder)
  • Each card: icon, name, description, status badge, "Launch" button
  • Later: this is also where onboarding will guide new users through setup

3. app.spawnkit.ai/office-executive/ ‚Äî Executive Theme (main dashboard)

Purpose: The actual product. Where you interact with your AI team.

What you see:

  • Office Grid ‚Äî 6 agent rooms arranged like an office floor plan
    • CEO Office (the boss ‚Äî you talk to this one)
    • Forge (CTO ‚Äî builds things, creates skills)
    • Atlas (COO ‚Äî operations, workflows)
    • Hunter (CRO ‚Äî research, revenue)
    • Echo (CMO ‚Äî content, communication)
    • Sentinel (QA ‚Äî quality, security)
    • Boardroom (active meetings/sessions)
  • Org Chart ‚Äî same team, shown as a hierarchy tree

Bottom bar panels (the control center):

  • Missions ‚Äî the team's task board. Active work, who's doing what, progress bars. Like a shared to-do list but the agents actually do the work. Inspired by Claude's Cowork view ‚Äî you should be able to clearly see what each agent is working on, their progress, and what's coming next
  • Crons ‚Äî scheduled jobs with live countdowns (morning briefs, auto-backups, etc.)
  • Memory ‚Äî the team's long-term memory (MEMORY.md). What they remember about you and past work
  • Chat ‚Äî talk to your team. Pick a target (CEO, or a remote fleet), send messages, see conversation history
  • Settings ‚Äî API keys, connection config
  • Remote ‚Äî other connected fleets. See who's online, send inter-fleet messages
  • Themes ‚Äî switch visual style without losing your data
  • Add Agent ‚Äî wizard to add new team members

CEO Office special features:

  • Inbox ‚Äî all communications in one place (Chat / Messages / Activity tabs)
  • Orchestration Center ‚Äî deep view into the team: agent list, skills, active sessions, missions, setup

Agent rooms ‚Äî click any room to see the agent detail panel: avatar, role, metrics (tokens used, last active), current task, sub-agents they've spawned, skills they have

4. Themed Variations (same data, different look)

  • Medieval ‚Äî CEO lives in a castle, Forge is a blacksmith, agents are knights. 3D scene with rivers, villages, towers
  • SimCity ‚Äî Your fleet as a living city. Isometric view, buildings = agents

These are "skins" on top of the same engine. All themes connect to the same data, same agents, same missions.

5. Onboarding Pages (planned/in progress)

Purpose: Guide new users from zero to a working fleet.

  • Step-by-step setup wizard
  • "What do you want your team to help with?" questionnaire
  • Auto-configure agents based on user's needs

6. fleet.spawnkit.ai ‚Äî Fleet Relay

Purpose: The "road system" connecting fleets together.

  • Not user-facing (works in the background)
  • REST API + WebSocket for real-time fleet-to-fleet communication
  • Office registration, heartbeats, messaging between fleets
  • Admin dashboard for monitoring

Built-in Superpowers (Skills)

Every SpawnKit team ships with two powerful skills that need friendlier names:

1. "Superpower: Brainstorm/Plan" (working name)

  • Deep thinking skill ‚Äî the team can brainstorm ideas, create plans, analyze problems
  • CEO uses this to break down your request into actionable steps
  • Needs a friendly name ‚Äî something like "MindStorm" or "ThinkTank" or "IdeaForge"

2. "Ralph Loop" (working name)

  • Iterative refinement ‚Äî the team reviews its own work, improves it, loops until quality is high
  • Automatic quality control built into every task
  • Needs a friendly name ‚Äî something like "PolishCycle" or "Refinery" or "PerfectLoop"

Adding new skills

  • Should feel like installing an app ‚Äî browse a catalog, click "Add", done
  • MCP (Model Context Protocol) connections for external tools
  • Skills marketplace planned ‚Äî share and discover skills from other fleets
  • In the themed version: you "go to the Forge" to create or install new skills

Known Issues (from prior testing)

  1. Settings "Loading..." on first click ‚Äî API bridge race condition (1s delay). Works on second click.
  2. Chat History stuck on "Loading..." ‚Äî endpoint may not return data correctly.
  3. Ghost test offices ‚Äî 9 test offices (Test1-9) pollute the Chat dropdown. Need cleanup in fleet relay.
  4. Static agent rooms ‚Äî Atlas/Forge/etc. show hardcoded data since only "main" agent exists in OpenClaw.
  5. Caddy config not permanent ‚Äî app.spawnkit.ai reverse_proxy was patched at runtime, reverts on restart.
  6. Onboarding not built yet ‚Äî new users currently land on the theme hub with no guidance.
  7. Skill names need work ‚Äî "superpower:brainstorm" and "ralph-loop" are internal names, need user-friendly alternatives.

What success looks like

A grandmother, a teenager, or someone who's never used an AI tool before should be able to:

  1. Land on spawnkit.ai and understand what it does in 10 seconds
  2. Click "Try SpawnKit" and arrive at a dashboard
  3. See their AI team working, understand what's happening
  4. Open the Inbox, send a message to their CEO agent
  5. See the team's task board and know what's being worked on
  6. Feel like they're managing a little company ‚Äî not wrestling with technology
