Critical UX Gaps

ğŸš¨ 1. No Onboarding Flow

A new user arrives at app.spawnkit.ai, sees a theme picker, clicks Executive Office, and gets... a fully populated office with 6 agents, none of which do anything. There is zero guidance on:

  • What SpawnKit is or does
  • How to connect their OpenClaw instance
  • What to do first
  • Why there are 6 rooms when they have 1 agent

This is the #1 thing blocking launch. A 3-step wizard (Connect â†’ Configure â†’ Go) would transform the product.

ğŸš¨ 2. Real vs. Decorative Confusion

The dashboard mixes real live data (sessions, crons, memory, chat) with decorative placeholders (5 agent rooms, skills list, messages tab, activity tab). A user cannot tell which parts are real. This erodes trust.

Fix: Every element should either show real data or explicitly say "Coming soon" with a visual distinction (greyed out, badge).

âš ï¸ 3. Navigation Model is Unclear

There are 3 overlapping navigation systems: (1) room clicks on the grid, (2) bottom status bar buttons, (3) Orchestration Center tabs. A user can find Sessions in the Orchestration panel but also expects session info when clicking an agent room. The mental model is fractured.

Fix: Rooms = agents. Clicking a room should show that agent's sessions, tasks, and chat. The Orchestration panel should be the "admin" view.
